## Why Portainer?

After managing Docker deployments for 20+ years, Portainer is the one tool I recommend to every IT team starting with containers. It turns Docker's command-line complexity into a clean web UI.

## Key Features

- **Visual Container Management** — Start, stop, restart, and monitor containers without touching the CLI
- **Stack Deployment** — Deploy Docker Compose stacks directly from the UI
- **Image Management** — Pull, build, and manage images visually
- **Multi-Environment** — Connect to local Docker, remote hosts, Kubernetes clusters
- **Access Control** — Team-based permissions for multi-user environments

## Self-Hosted Setup

```bash
docker volume create portainer_data
docker run -d -p 9443:9443 --name portainer \
  --restart=always \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v portainer_data:/data \
  portainer/portainer-ce:latest
```

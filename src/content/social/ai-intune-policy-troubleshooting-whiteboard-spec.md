# Drawy Whiteboard Spec — AI Intune Policy Troubleshooting

## Goal
Create one whiteboard-style social visual summarizing the article:
**How to Use AI to Triage Intune Policy Failures Faster**

## Format
- Aspect ratio: 1:1 (1080x1080)
- Style: hand-drawn whiteboard / sketchnote
- Tone: practical, technical, clear hierarchy
- Brand feel: desktop engineering + Microsoft endpoint operations

## Core Message
"AI Intune policy troubleshooting is fastest when AI ranks hypotheses and engineers validate before remediation."

## Visual Layout

### Header block
- Title: "AI Intune Policy Triage — Faster, Safer"
- Subheader: "AI suggests. Engineers verify."

### Center flow (7 steps)
Draw a left-to-right or top-to-bottom flow with arrows:
1. Scope failure
2. Collect evidence bundle
3. Redact sensitive data
4. Prompt for ranked hypotheses
5. Run deterministic checks
6. Choose low-risk remediation path
7. Document + reuse runbook

### Side callouts
- "Do": structured prompts, confidence labels, validation matrix
- "Don’t": unredacted logs, blind config changes, autopilot fixes

### Bottom metrics strip
- "Save 20–45 min / incident"
- "Improve MTTR"
- "Reduce repeat incidents"

## Icon Suggestions
- Magnifier for scope
- Folder stack for evidence
- Shield for redaction
- Robot/chat bubble for AI prompt
- Checklist for validation
- Wrench for remediation
- Notebook for runbook

## Color/Styling Notes
- White background
- Black marker outlines
- Accent colors: blue + green + orange
- Keep text bold, minimal, legible on mobile
